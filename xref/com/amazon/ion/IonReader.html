26  /* One design goal is for the readers and writers to be independent of an
27   * IonSystem or ValueFactory and thus independent of particular implementations
28   * of the DOM.
29   *
30   * The issue is that one needs a ValueFactory in order to construct the tree.
31   * So one either needs to pass a ValueFactory / IonSystem to the reader, or
32   * pass the reader to the system.  I decided that the dependencies were better
33   * the latter way.  So we have IonSystem.newValue(IonReader) instead of
34   * IonReader.nextValue(IonSystem).
35   */

37  /**
38   * Provides stream-based access to Ion data independent of its underlying
39   * representation (text, binary, or {@link IonValue} tree).
40   * <p>
41   * <b>WARNING:</b> This interface should not be implemented or extended by
42   * code outside of this library.
43   * We still have some work to do before this interface is stable.
44   * See <a href="https://github.com/amzn/ion-java/issues/11">issue amzn/ion-java/issues/11</a>
45   * <p>
46   * An {@code IonReader} has a "cursor" tracking the <em>current value</em> on
47   * which the reader is positioned. Generally, newly created readers are not
48   * positioned on any value. To begin traversing the Ion data, one would use
49   * {@link #next()} to advance the cursor onto the first value (or learn there isn't
50   * one). Once positioned, the current value's data can be accessed with the
51   * {@code *Value()} methods.
52   * <p>
53   * When the current value is a container, calling {@link #next()} moves the
54   * cursor to the <em>next sibling</em> of the container, at the same depth,
55   * skipping over any children the container may have.
56   * To read the children, call {@link #stepIn()},
57   * then {@link #next()} to position onto the first child value (or learn there
58   * isn't one).  Calling {@link #stepOut()} skips over any remaining children
59   * and moves the cursor just beyond the container; call {@link #next()} to
60   * move the cursor to the following value.
61   * <p>
62   * In general, method names are intended to parallel similar methods in the
63   * {@link IonValue} hierarchy.  For example, to get the text of a symbol one
64   * would use {@link #stringValue()}, mirroring {@link IonSymbol#stringValue()}.
65   *
66   * <h2>Exception Handling</h2>
67   * {@code IonReader} is a generic interface for traversing Ion data, and it's
68   * not possible to fully specify the set of exceptions that could be thrown
69   * from the underlying data source.  Thus all failures are thrown as instances
70   * of {@link IonException}, wrapping the original cause.  If an application
71   * wants to handle (say) {@link IOException}s specially, then it needs to
72   * extract that from the wrappers; the documentation of {@link IonException}
73   * explains how to do that.
74   *
75   * <h2>Reader Facets</h2>
76   * Readers are {@link Faceted} and implementations may provide additional
77   * functionality accessible via the {@link #asFacet(Class)} method.
78   *
79   * <h3>The {@link SpanProvider} Facet</h3>
80   * This facet is available on all readers that directly consume an Ion source.
81   * It provides access to the "{@linkplain SpanProvider#currentSpan() current
82   * span}" covering the reader's current value.
83   * There is <em>not</em> a current span at the start of the source, immediately
84   * after a call to {@link #stepIn()} or {@link #stepOut()}, or when the prior
85   * call to {@link #next()} returned null (meaning: end of container or end of
86   * stream). In such states, {@link SpanProvider#currentSpan()} will fail.
87   *
88   * <h3>The {@link SeekableReader} Facet</h3>
89   * This facet is available on all readers <em>except</em> those created from
90   * an {@link java.io.InputStream InputStream}.
91   * (See <a href="https://github.com/amzn/ion-java/issues/17">issue amzn/ion-java/issues/17</a>.)
92   * It allows the user to reposition the reader to a {@link Span} over the
93   * same reader instance or another reader with the same source.
94   *
95   * <h2>Span Facets</h2>
96   * Readers that support the {@link SpanProvider} facet vend {@link Span}s that
97   * are also faceted.
98   *
99   * <h3>The {@link OffsetSpan} Facet</h3>
100  * This facet is support by all readers of Ion binary and text data.
101  *
102  * <h3>The {@link TextSpan} Facet</h3>
103  * This facet is supported by all readers of Ion text data.
104  */
105 public interface IonReader
106     extends Closeable, Faceted
107 {
108 
109     /**
110      * Determines whether there is another value at the current depth;
111      * in other words whether there is a sibling value that may be reached
112      * using {@link #next()}.
113      * This method may be
114      * called multiple times, which does not move the current position.
115      * <p>
116      * <b>WARNING:</b> this method alters the internal state of the reader such
117      * that you cannot reliably get values from the "current" element. The only
118      * thing you should call after {@code hasNext()} is {@link #next()}!
119      *
120      * @deprecated Applications should detect the end of the current level by
121      * checking for a {@code null} response from {@link #next()}.
122      */
123     @Deprecated
124     public boolean hasNext();
125 
126     /**
127      * Positions this reader on the next sibling after the current value,
128      * returning the type of that value.  Once so positioned the contents of
129      * this value can be accessed with the {@code *Value()} methods.
130      * <p>
131      * A sequence of {@code next()} calls traverses the data at a constant
132      * depth, within the same container.
133      * Use {@link #stepIn()} to traverse down into any containers, and
134      * {@link #stepOut()} to traverse up to the parent container.
135      *
136      * @return the type of the next Ion value (never {@link IonType#DATAGRAM}),
137      * or {@code null} when there are no more elements at the current depth in
138      * the same container.
139      */
140     public IonType next();
141 
142 
143     /**
144      * Positions the reader just before the contents of the current value,
145      * which must be a container (list, sexp, or struct).
146      * There's no current value immediately after stepping in, so the next
147      * thing you'll want to do is call {@link #next()} to move onto the first
148      * child value (or learn that there's not one).
149      * <p>
150      * Stepping into a null container ({@code null.list}, {@code null.sexp},
151      * or {@code null.struct}) behaves as if the container were empty
152      * ({@code []}, {@code ()}, or <code>{}</code>).
153      * <p>
154      * At any time {@link #stepOut()} may be called to move the cursor back to
155      * (just after) the parent value, even if there's more children remaining.
156      *
157      * @throws IllegalStateException if the current value isn't an Ion container.
158      */
159     public void stepIn();
160 
161     /**
162      * Positions the iterator after the current parent's value, moving up one
163      * level in the data hierarchy.
164      * There's no current value immediately after stepping out, so the next
165      * thing you'll want to do is call {@link #next()} to move onto the
166      * following value.
167      *
168      * @throws IllegalStateException if the current value wasn't stepped into.
169      */
170     public void stepOut();
171 
172     /**
173      * Returns the depth into the Ion value that this reader has traversed.
174      * At top level the depth is 0, and it increases by one on each call to
175      * {@link #stepIn()}.
176      */
177     public int getDepth();
178

182      * This may be either a system or local symbol table.
183      */

188      * Returns the type of the current value, or null if there is no

192 
193     /**
194      * Returns an {@link IntegerSize} representing the smallest-possible
195      * Java type of the Ion {@code int} at the current value.
196      *
197      * If the current value is {@code null.int} or is not an Ion
198      * {@code int}, or if there is no current value, {@code null} will
199      * be returned.
200      *
201      * @see IonInt#getIntegerSize()
202      */
203     public IntegerSize getIntegerSize();
204 
205     /**
206      * Return the annotations of the current value as an array of strings.
207      *
208      * @return the (ordered) annotations on the current value, or an empty
209      * array (not {@code null}) if there are none.
210      *
211      * @throws UnknownSymbolException if any annotation has unknown text.
212      */
213     public String[] getTypeAnnotations();
214 
215     /**
216      * Gets the current value's annotations as symbol tokens (text + ID).
217      *
218      * @return the (ordered) annotations on the current value, or an empty
219      * array (not {@code null}) if there are none.
220      *
221      */
222     public SymbolToken[] getTypeAnnotationSymbols();
223 
224     /**
225      * Return the annotations on the curent value as an iterator.  The
226      * iterator is empty (hasNext() returns false on the first call) if
227      * there are no annotations on the current value.
228      *
229      * Implementations *may* throw {@link UnknownSymbolException} from
230      * this method if any annotation contains unknown text. Alternatively,
231      * implementations may provide an Iterator that throws
232      * {@link UnknownSymbolException} only when the user navigates the
233      * iterator to an annotation with unknown text.
234      *
235      * @return not null.
236      */
237     public Iterator<String> iterateTypeAnnotations();

241     /**
242      * Gets the symbol ID of the field name attached to the current value.
243      * <p>
244      * <b>This is an "expert method": correct use requires deep understanding
245      * of the Ion binary format. You almost certainly don't want to use it.</b>
246      *
247      * @return the symbol ID of the field name, if the current value is a
248      * field within a struct.
249      * If the current value is not a field, or if the symbol ID cannot be
250      * determined, this method returns a value <em>less than one</em>.

252      * @deprecated Use {@link #getFieldNameSymbol()} instead.
253      */
254     @Deprecated

256 
257     /**
258      * Return the field name of the current value. Or null if there is no valid
259      * current value or if the current value is not a field of a struct.
260      *
261      * @throws UnknownSymbolException if the field name has unknown text.
262      */
263     public String getFieldName();
264 
265     /**
266      * Gets the current value's field name as a symbol token (text + ID).
267      * If the text of the token isn't known, the result's
268      * {@link SymbolToken#getText()} will be null.
269      * If the symbol ID of the token isn't known, the result's
270      * {@link SymbolToken#getSid()} will be
271      * {@link SymbolTable#UNKNOWN_SYMBOL_ID}.
272      * At least one of the two fields will be defined.
273      *
274      * @return null if there is no current value or if the current value is
275      *  not a field of a struct.
276      *
277 
278      */
279     public SymbolToken getFieldNameSymbol();

282     /**
283      * Determines whether the current value is a null Ion value of any type
284      * (for example, <code>null</code> or <code>null.int</code>).
285      * It should be called before
286      * calling getters that return value types (int, long, boolean,
287      * double).
288      */
289     public boolean isNullValue();

291     /**
292      * Determines whether this reader is currently traversing the fields of an
293      * Ion struct. It returns false if the iteration
294      * is in a list, a sexp, or a datagram.

296     public boolean isInStruct();
