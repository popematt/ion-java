167     /**
168      * <p>
169      * Determines whether the IonReader will allow incremental reading of binary Ion data. When enabled, if
170      * {@link IonReader#next()} returns {@code null} at the top-level, it indicates that there is not enough data
171      * in the stream to complete a top-level value. The user may wait for more data to become available in the stream
172      * and call {@link IonReader#next()} again to continue reading. Unlike the non-incremental reader, the incremental
173      * reader will never throw an exception due to unexpected EOF during {@code next()}. If, however,
174      * {@link IonReader#close()} is called when an incomplete value is buffered, an {@link IonException} will be raised.
175      * </p>
176      * <p>
177      * There is currently no incremental text IonReader, so for text data a non-incremental IonReader will be
178      * returned regardless of the value of this option. If incremental text reading is supported in the future, it
179      * may be enabled via this option.
180      * </p>
181      * <p>
182      * When this option is enabled, auto-detection of GZIP data is not supported; the byte array or InputStream
183      * implementation provided to {@link #build(byte[])} or {@link #build(InputStream)} must return uncompressed bytes.
184      * This can be achieved by wrapping the data in a GZIPInputStream and passing it to {@link #build(InputStream)}.
185      * </p>
186      * <p>
187      * Additionally, when this option is enabled, annotation iterators are reused by default, improving performance.
188      * See {@link #withAnnotationIteratorReuseEnabled(boolean)} for more information and to disable that option.
189      * </p>
190      * <p>
191      * Although the incremental binary IonReader provides performance superior to the non-incremental binary IonReader
192      * for both incremental and non-incremental use cases, there is one caveat: the incremental implementation
193      * must be able to buffer an entire top-level value and any preceding system values (Ion version marker(s) and
194      * symbol table(s)) in memory. This will not be a problem for the vast majority of Ion streams, as it is rare for a
195      * single top-level value or symbol table to exceed a few megabytes in size. However, if the size of the stream's
196      * values risk exceeding the available memory, then this option must not be enabled.
197      * </p>
198      * @param isEnabled true if the option is enabled; otherwise, false.
199      *
200      * @return this builder instance, if mutable;
201      * otherwise a mutable copy of this builder.
202      *
203      * @see #setIncrementalReadingEnabled()
204      * @see #setIncrementalReadingDisabled()
205      */
206     public IonReaderBuilder withIncrementalReadingEnabled(boolean isEnabled) {
207         IonReaderBuilder b = mutable();
208         if (isEnabled) {
209             b.setIncrementalReadingEnabled();
210         } else {
211             b.setIncrementalReadingDisabled();
212         }
213         return b;
214     }

276     /**
277      * <p>
278      * Determines whether readers will reuse the annotation iterator returned by
279      * {@link IonReader#iterateTypeAnnotations()}. When enabled, the returned iterator remains valid only while the
280      * reader remains positioned at the current value; storing the iterator and iterating its values after that will
281      * cause undefined behavior. This provides improved performance and memory efficiency when frequently iterating
282      * annotations. When disabled, the returned iterator may be stored and used to retrieve the annotations that were
283      * on the value at the reader's position at the time of the call, regardless of where the reader is currently
284      * positioned.
285      * </p>
286      * <p>
287      * Currently, this option only has an effect when incremental reading is enabled (see
288      * {@link #withIncrementalReadingEnabled(boolean)}). In that case, it is enabled by default. Non-incremental readers
289      * always act as if this option were disabled.
290      * </p>
291      * @param isEnabled true if the option is enabled; otherwise, false.
292      *
293      * @return this builder instance, if mutable;
294      * otherwise a mutable copy of this builder.
295      *
296      * @see #setAnnotationIteratorReuseEnabled()
297      * @see #setAnnotationIteratorReuseDisabled()
298      */
